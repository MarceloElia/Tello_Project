## What this project is

A DJI Tello drone controller that accepts hand gestures (MediaPipe + webcam) and spoken
commands (Whisper → Ollama → JSON) — all running 100% locally. A single
`DroneController` abstraction switches between a software mock, a PyBullet physics sim,
and the real drone without changing any control code.

## Stack

- Python 3.11+
- OpenCV — gesture camera window and overlay rendering
- MediaPipe — hand landmark detection and gesture classification
- faster-whisper — local speech-to-text
- Ollama / qwen2.5:3b — local LLM for voice command parsing (note: Aider uses qwen2.5-coder:3b)
- PyBullet / gym-pybullet-drones — quadrotor physics simulation
- djitellopy — real Tello SDK over Wi-Fi
- pytest — hardware-free test suite
- conda (tello-sim env) + venv (main env)

### Async / multi-threaded logic
Why: `AsyncCommandRunner` and `ThreadedCtrlAdapter` in `gesture/runner.py` use queues
and locks. Debugging or extending requires tracking shared mutable state across threads.
Examples:
- "Why does the gesture command queue block the camera feed?"
- "Add a priority lane to AsyncCommandRunner for emergency commands"

### Cross-module feature additions
Why: A new gesture end-to-end touches `detector.py` (enum), `command_map.py` (mapping),
and `app.py` (label) — three files with inter-dependencies Aider will likely get wrong.
Examples:
- "Add a new two-finger swipe gesture for rotation"
- "Add a HOVER_HIGH gesture that flies to 150 cm"

### PyBullet physics and PID changes
Why: `pybullet_backend.py` couples MockTello's logic with PyBullet's physics loop,
cooperative-mode tick scheduling, and PID control — requires understanding the full
inheritance chain.
Examples:
- "The drone overshoots when going up — fix the PID"
- "Make the camera follow smoother"

### Voice pipeline (STT → LLM → validation)
Why: `stt.py`, `llm_parser.py`, `listener.py`, and `commands.py` form a multi-file
async chain. Any change needs context from all four simultaneously.
Examples:
- "The wake-word detector triggers on background noise"
- "Add a new validated command type"

### New test cases
Why: Tests validate the MockTello safety layer and debounce logic together.
Writing correct tests requires understanding invariants across multiple files.
Examples:
- "Write a test for the new gesture mapping"
- "Test that emergency stop works mid-queue"

### Architecture and backend integration
Why: Changes that must work across mock, sim, and real require understanding the full
abstraction stack and PyBulletBackend's override chain.
Examples:
- "Add a hover() method to DroneController"
