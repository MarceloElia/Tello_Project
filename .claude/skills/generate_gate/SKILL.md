You are analysing this project to create a Task Gate for **qwen2.5-coder:3b running via
Aider** — a local coding agent (~7B parameters, ~4.7 GB RAM) the developer runs offline

Unlike a pure chat model, Aider has real file access and produces diffs — so the gate
should reflect tasks where a 7B coder model can reliably read one file, make a focused
change, and produce a correct diff.

## Step 2 — Analyse qwen2.5-coder:3b + Aider fit

**Aider capabilities with a 7B coder model:**
- Docstrings, type hints, and inline comments for single functions
- Constant and magic-number edits
- Display/label string changes
- Adding a single entry to a dict or list following a clear existing pattern
- Writing a short function body from a stub with a clear signature and docstring
- Simple isolated bug fixes (wrong value, off-by-one) in a single function
- Variable/function renaming within one file
- Appending structured content to docs/notes files
- Basic regex and string manipulation in one file

**Hard limits — always hold for Claude:**
- Multi-file changes where files have inter-dependencies
- Async, concurrent, or multi-threaded logic
- Architecture or design decisions
- Debugging across more than one file
- Writing full features from scratch
- Test suites that require understanding cross-file invariants
- Any change that must work across multiple backends/environments simultaneously
